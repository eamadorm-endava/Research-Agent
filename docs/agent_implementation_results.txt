Implemented the Drive OAuth feature end to end in the Research-Agent codebase.

What changed

Enabled the agent to register Google Drive as an OAuth-enabled McpToolset instead of leaving it commented out.
Centralized MCP tool construction so BigQuery, GCS, and Drive are built consistently from config.
Added a safe gate for Drive: if DRIVE_OAUTH_CLIENT_ID or DRIVE_OAUTH_CLIENT_SECRET is missing, the agent skips the Drive MCP tool instead of exposing a broken tool that cannot trigger OAuth.
Preserved the dual-auth model for Drive:
User OAuth token goes in Authorization: Bearer <access_token> via ADK/Gemini Enterprise.
Cloud Run service auth still uses X-Serverless-Authorization when needed.
Added a Gemini Enterprise runbook covering OAuth client setup, authorization resource creation, delete/re-register flow, validation, and troubleshooting.
