As a developer,
I want to build a dedicated MCP server using the google-cloud-bigquery Python client,
So that the MCP server has a modular, reusable, and secure way to interact with BigQuery to manage datasets, read data, and insert records.

Acceptance Criteria

These conditions must be met for the feature to function as requested:

Folder Structure

All code for this connector must be developed and contained within the Research-Agent/mcp_servers/big_query directory at the root of the repository.

Client Initialization

The connector must initialize the BigQuery client seamlessly (relying on Google Application Default Credentials so it works locally via impersonation and in the cloud).

Connector Capabilities

The mcp server (a Python class) must implement methods to handle the following operations:

Create: Ability to create new Datasets and Tables.
Read: Execute read-only SQL queries and safely return the results (allowing to keep track of the complex columns such as structs, arrays, and its combinations).
Get Table Schema: Retrieve and return the schema definition of a specific table.
List Tables: Retrieve a list of all tables within a given dataset.
Add Row: Insert a new row (or multiple rows) into an existing table.

Definition of Done

Code is fully typed (using Python type hints) and includes clear docstrings for every method.
When necessary, use data types directly from the google-bigquery library (for example: google.cloud.bigquery.schema.SchemaField)
Code passes standard formatting and linting checks (e.g., using ruff, black, or your project's chosen linter).
Unit tests are implemented for the connector methods (mocking the BigQuery client to avoid actual cloud calls during CI).
A basic README.md or usage example is added inside /mcp_servers/big_query to show how to instantiate and call the connector.
The PR is successfully reviewed and merged into the main development branch.
