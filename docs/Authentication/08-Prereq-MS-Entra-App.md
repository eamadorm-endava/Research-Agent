# Prerequisite 2: Microsoft Entra ID App Registration

To allow Google Cloud's Workforce Identity Pool to verify tokens generated by Microsoft Entra ID, you must register an application within your Microsoft Entra ID (formerly Azure AD) tenant. 

This app registration acts as the "client" that Google Cloud will trust.

## Why do you need an App Registration?

When a user logs into your portal (e.g., your Gemini Enterprise frontend), they authenticate against Microsoft Entra ID. Entra ID issues an OIDC token that proves their identity. Google Cloud needs a way to verify that this token is legitimate and was issued for the correct purpose. 

By registering an App in Entra ID, you obtain a **Client ID** and a **Tenant ID**, which you provide to Google Cloud so it knows exactly which tokens to trust.

## How to Register the App in Microsoft Entra ID

Follow these steps in the Microsoft Azure Portal:

### 1. Register a new application
1. Sign in to the [Microsoft Azure Portal](https://portal.azure.com/).
2. Navigate to **Microsoft Entra ID** > **App registrations**.
3. Click **New registration**.
4. Enter a name (e.g., `Google Cloud Workforce Federation`).
5. Under **Supported account types**, select who can use the application (usually *Accounts in this organizational directory only*).
6. Click **Register**.

### 2. Gather the Required Information for CLI
After the app is registered, you will be taken to its **Overview** page. You need to copy two pieces of information to use in the Google Cloud CLI script:

*   **Application (client) ID:** This maps to the `--client-id` parameter in the `gcloud` command.
*   **Directory (tenant) ID:** This is used to build the `--issuer-uri` parameter.

### 3. Construct the Issuer URI
Google Cloud needs the OIDC issuer URI to know where to fetch Microsoft's public keys for token validation. 

For Microsoft Entra ID v2.0 endpoints, the Issuer URI format is:
```text
https://login.microsoftonline.com/YOUR_TENANT_ID/v2.0
```
*(Replace `YOUR_TENANT_ID` with the Directory (tenant) ID you copied above).*

## Summary of CLI Parameters

When you run the `define_identity_provider.sh` script, you will map the information gathered here as follows:

| Microsoft Entra ID Value | Google Cloud CLI Parameter (`gcloud`) |
| :--- | :--- |
| Application (client) ID | `--client-id="YOUR_CLIENT_ID"` |
| Directory (tenant) ID | `--issuer-uri="https://login.microsoftonline.com/YOUR_TENANT_ID/v2.0"` |

Once this is set up, Google Cloud is fully equipped to validate tokens issued by this Microsoft Entra ID application.
